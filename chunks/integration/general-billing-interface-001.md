# The General Billing Interface in SAP SD

## Operational Summary
The *General Billing Interface* is a traditional user exit mechanism that allows external (non-SAP or legacy) orders and deliveries to be billed inside SAP SD. The external data must be prepared in a defined sequential file format with required fields populated. Optional fields can be filled either from the incoming data or through standard SAP determination logic. External reference numbers can be preserved in the billing document for traceability.

## Questions This Chunk Answers
- How can orders originating outside SAP be billed inside the SAP system?
- What file format is required for the general billing interface?

## What It Integrates
The general billing interface connects **external order/delivery management systems** (non-SAP or legacy systems) with **SAP SD Billing**. It enables these external transactions to generate SAP billing documents and FI postings, as if they had originated in the SAP order-to-cash process.

## Involved Modules or Systems
- **External systems**: Legacy order management, non-SAP ERP, third-party fulfillment platforms
- **SAP SD Billing**: Processes the interface data and creates billing documents
- **SAP FI**: Receives the accounting postings generated by the billed documents

## Affected SAP Objects
- SD billing document (created as output of the interface)
- FI accounting document (created when the billing document is released)

## Data Flow

| Source | Object / Data | Destination | When |
|---|---|---|---|
| External system | Sequential file with order/delivery data | SAP SD Billing (via user exit) | At billing run |
| Sequential file | Required fields (customer master, sales org) | SD Billing document header | At processing |
| Sequential file or SAP determination | Optional fields (material, price components) | SD Billing document items | At processing |
| External system | External reference numbers | Billing document reference fields | At processing |
| SD Billing | Billing document | FI accounting document | At release to accounting |

## Relevant Configuration Points
- User exit must be implemented and activated to read and process the sequential file.
- Required fields in the interface file: at minimum the customer master number and sales organization.
- Optional fields can be sourced from the file or delegated to standard SAP determination (pricing, material determination, etc.).
- External reference numbers can be mapped to the billing document's reference fields for downstream traceability.

## Functional Impact
External transactions billed through this interface generate the same FI accounting documents as native SAP billing: G/L postings for revenue, receivables, and VAT. The billing documents appear in standard SAP billing reports and are subject to the same account determination and output management rules.

## Common Integration Errors

**Interface file rejected — required field missing**
→ The sequential file is missing the customer master number or sales organization. Both are mandatory; without them SAP cannot determine the company code or account determination group.

**Billing document created but account determination fails**
→ Account assignment groups (payer or material) or the chart of accounts assignment is not configured for the data in the external file. Check account determination Customizing.
